üçé macOS
üîë
System-wide (Safari, Chrome, Edge, Brave)

GUI Method

  1. Download the cacert.crt file
  2. Double-click the file to open Keychain Access
  3. Select System keychain if prompted
  4. Enter your admin password when prompted
  5. Find the certificate in Keychain Access and double-click it
  6. Expand Trust section
  7. Set When using this certificate to Always Trust
  8. Close the window and enter your password to save changes

Command Line Method

  1. Open Terminal
  2. Run: sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain cacert.crt
  3. Enter your admin password when prompted
Note: On newer macOS versions, you may need to use trustAsRoot instead of trustRoot.

üêß Linux
üíª
System-wide (Chrome, Edge, Brave)

Debian/Ubuntu

  1. Download the certificate: wget https://relaxdiego.com/ca-cert/cacert.crt
  2. Create CA directory (optional): sudo mkdir -p /usr/local/share/ca-certificates/my-custom-ca
  3. Copy to system certificates: sudo cp cacert.crt /usr/local/share/ca-certificates/my-custom-ca/
  4. Update certificates: sudo update-ca-certificates
  5. Restart your browser

RHEL/CentOS

  1. Download the certificate: wget https://relaxdiego.com/ca-cert/cacert.crt
  2. Copy to trust anchors: sudo cp cacert.crt /etc/pki/ca-trust/source/anchors/
  3. Update trust store: sudo update-ca-trust
  4. Restart your browser

üì± iOS
üì≤
Safari (System-wide)
  1. Download the certificate using Safari (ensure file has .crt extension)
  2. When prompted, tap Allow to download the profile
  3. Go to Settings > General > VPN & Device Management
  4. Under Downloaded Profile, tap the certificate
  5. Tap Install and enter your passcode
  6. Tap Install again to confirm
  7. CRITICAL: Go to Settings > General > About > Certificate Trust Settings
  8. Under "Enable full trust for root certificates," toggle ON your certificate
  9. Tap Continue to confirm the security warning
Important: The certificate must be manually trusted in Certificate Trust Settings. Without this step, the certificate will not be trusted for SSL/TLS connections.
